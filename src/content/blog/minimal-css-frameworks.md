In the last five years, we've seen a massive surge in utility-first frameworks. They solved the problem of naming things, but they introduced a new problem: readability of the HTML itself.

As I build more systems, I find myself craving the simplicity of a single CSS file with well-defined variables and a few reusable classes.

### The Return of Semantic CSS

Is it time to go back? Not entirely. But using standard CSS features like `@scope`, nesting, and container queries makes many of the reasons we used frameworks obsolete.

I'm currently experimenting with a "blueprint" style—something that feels technical and structured without the overhead of a massive design system. This very site is a result of that experimentation.

### Final Thoughts

The best framework is the one that stays out of your way. Sometimes, that's no framework at all.
